## Слайд 2 — Проблема

**Комплаенс тонет в ложных тревогах**

Буллеты:
- 90–95% срабатываний AML-мониторинга — ложные (false positive)
- Правила ловят одну транзакцию и слепы к схемам из цепочки счетов и времени
- Аналитик часами вручную «разматывает» граф по каждому алерту
- Чёрные ящики не дают обоснования, пригодного для SAR-отчёта

✏️ СОБРАТЬ: по центру огромная цифра «90–95%» с подписью «ложных срабатываний»; рядом мелкая иконка аналитика под грудой алертов. Файла нет — это текст+иконка средствами PowerPoint.

Заметки: AML-мониторинг генерирует поток тревог, из которых 90–95% ложные. Классические правила видят отдельный платёж, но не схему, растянутую по счетам и времени. Дорогой аналитик разбирает граф вручную, а модели-чёрные-ящики не дают формулировки для отчёта регулятору.

## Слайд 3 — Инсайт и идея решения  (объединённый)

**Отмывание — это ПАТТЕРН на графе. Ловим мотивы, а затем объясняем**

Буллеты:
- Fan-out на дропов, циклы, смёрфинг, gather-scatter — узнаваемые мотивы
- Отдельная транзакция легальна — подозрительна форма цепочки
- Решение = 3 блока: (1) майнинг мотивов → (2) GNN-скоринг → (3) LLM-объяснитель
- Итог для аналитика: цепочка + готовое SAR-обоснование

🖼 ФАЙЛ: `outputs/figures/fig02_motif_freq.png` — частоты мотивов среди нелегальных цепочек (левая половина слайда).
✏️ СОБРАТЬ (правая половина): горизонтальная схема-конвейер из 3 блоков:
   «Граф транзакций → [Мотивы ⊕ GNN] → скор рёбер → [LLM-агент] → SAR-нарратив».

Заметки: Главная идея: отмывание оставляет след в форме графа — веер на дропов, кольца, дробление, сбор-и-разброс. По отдельности транзакция чиста, подозрительна конфигурация. Поэтому наш конвейер: явные мотивы дают понятные признаки, лёгкий GNN добавляет обучаемый скор, а LLM-агент превращает цепочку в человекочитаемое обоснование для SAR.

## Слайд 5 — Методы и метрики

**Меряем то, что важно при доле класса <1%**

Буллеты:
- Minority-F1 и PR-AUC — НЕ accuracy/ROC-AUC (при дисбалансе они лгут)
- Precision@k, recall по каждому мотиву, alert-reduction при фикс. recall
- Строгий темпоральный сплит: train 70% по времени → нет утечки
- Три модели: мотивы+LightGBM (baseline) · GNN (learned) · Random (контроль)

✏️ СОБРАТЬ: две панели. Слева — «НЕ меряем: Accuracy, ROC-AUC» (перечёркнуто). Справа — «Меряем: minority-F1, PR-AUC, precision@k, alert↓». Внизу — стрелка времени с разрезом train | test. Файла нет.

Заметки: При доле класса меньше процента accuracy и ROC-AUC вводят в заблуждение — смотрим minority-F1, PR-AUC, precision@k и снижение алертов при фиксированном recall. Обучение и тест разделены строго по времени, это исключает утечку из будущего. Сравниваем интерпретируемый бейзлайн, GNN и случайный контроль.

## Слайд 6 — Результаты

**Интерпретируемый бейзлайн — сильнейший**

Буллеты:
- LightGBM (мотивы): minority-F1 = 0,42 · PR-AUC = 0,38
- GNN (обучаемый): F1 = 0,25 · PR-AUC = 0,19 — конкурентен, есть куда расти
- Random (контроль): F1 = 0,06 · PR-AUC = 0,03
- P@100 = 100% у LightGBM; тест 75 000 рёбер, 2 304 нелегальных

🖼 ФАЙЛ: `outputs/figures/fig03_pr_curve.png` — PR-кривые трёх моделей, LightGBM выше всех.

Заметки: Честный результат: сильнее всего оказались интерпретируемые мотив-признаки с LightGBM — F1 около 0,42, PR-AUC 0,38, это в разы выше случайного контроля. Простой GNN пока отстаёт (F1 ~0,25); показываем его как полноценную обучаемую модель с планом развития, но он не превосходит бейзлайн. Именно объяснимый бейзлайн — наша основа.

## Слайд 7 — Импакт

**Часы ручного разбора → секунды**

Буллеты:
- Снижение объёма алертов на 91,9% при сохранении recall 50%
- На разбор уходит лишь ~8% потока против сплошной ручной проверки
- Precision@50 и Precision@100 = 100% — верх списка чистый
- Recall по мотивам: scatter-gather 0,66 · fan-in 0,65 · gather-scatter 0,58

🖼 ФАЙЛ: `outputs/figures/fig04_precision_at_k.png` — precision@k, LightGBM держит 1,0 на топе.
✏️ СОБРАТЬ (поверх/рядом): крупная плашка «−91,9% алертов».

Заметки: Отсюда бизнес-эффект. При фиксированном recall 50% модель срезает поток тревог почти на 92%, аналитику остаётся разобрать порядка 8%. При этом первые 50 и 100 кандидатов — стопроцентно релевантны, время тратится не впустую. Часы ручной размотки графа превращаются в секунды приоритизации.

## Слайд 9 — Инженерия и продукт  (объединённый)

**Воспроизводимо, и понятно кому продаём**

Буллеты:
- MLOps: Git + Docker + CI (тесты/линт) + DVC (данные) + MLflow (эксперименты)
- Любой прогон повторяется одной командой → те же метрики и фигуры
- Покупатель: комплаенс/риск банков, платёжных систем, крипто-бирж (подают SAR)
- Отличие от вендоров (Feedzai, Quantexa, Actimize) и OSS (Multi-GNN): явные мотивы + трассировка + авто-нарратив
- Почему сейчас: EU AMLA / FATF — объяснимый AI стал требованием

✏️ СОБРАТЬ: слева мини-схема пайплайна «Data (DVC) → Train (Docker) → CI → MLflow → outputs/»; справа матрица 2×2 «Объяснимость × Открытость», AML-GRAPH в правом верхнем углу. Файлов нет.

Заметки: Проект собран как воспроизводимый пайплайн: код в Git, окружение в Docker, данные под DVC, эксперименты в MLflow, CI гоняет тесты на каждый коммит. Покупатель — комплаенс-подразделения, обязанные подавать SAR. Закрытые вендоры дороги и непрозрачны, академический OSS не даёт нарратива — наша ниша ровно посередине, и регуляторы сделали объяснимость обязательной.
